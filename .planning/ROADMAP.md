## Phase Details

### Phase 1: Visual Foundation & Performance
**Goal:** Site has a distinctive, premium visual identity that feels friendly/local while achieving Core Web Vitals targets for SEO ranking.

**Depends on:** Nothing (first phase)

**Requirements:** DESIGN-01, DESIGN-02, DESIGN-03, DESIGN-04, DESIGN-05, DESIGN-06, SEO-01, SEO-09, SEO-10

**Success Criteria** (what must be TRUE):
1. User sees a visually distinctive site that doesn't look like generic contractor templates
2. User encounters real project photos (not stock images) on hero, service pages, and gallery
3. User experiences smooth micro-interactions that reinforce the "Breeze" brand (cloud animations, transitions)
4. All pages load with LCP <2.5s on 3G mobile connection (Core Web Vitals passing)
5. User perceives coastal NC visual storytelling through imagery and local references

### Phase 2: Conversion Architecture
**Goal:** Every page drives user toward phone call or estimate request within 60 seconds through dual CTA strategy and persistent trust signals.

**Depends on:** Phase 1 (visual foundation must exist for CTA placement)

**Requirements:** CONV-01, CONV-02, CONV-03, CONV-04, CONV-05, CONV-06, CONV-07, CONV-08

**Success Criteria** (what must be TRUE):
1. User can tap to call from any page (mobile: sticky bottom bar, desktop: header button)
2. User can reach estimate request form within 1 click from any page
3. User sees trust signals on every page: 4.9 Google rating, review count, Fortified certification, license number
4. User on /estimate page can use Roofr instant estimator with proper mobile UX
5. User sees warranty/guarantee information and expected project timeline on service pages
6. User encounters 24/7 emergency service CTA prominently on all pages

### Phase 3: SEO Technical Foundation
**Goal:** Site achieves perfect technical SEO with comprehensive schema markup, optimized metadata, and proper internal linking for maximum crawlability and rich results eligibility.

**Depends on:** Phase 1 (performance foundation) and Phase 2 (CTA structure for internal linking)

**Requirements:** SEO-02, SEO-03, SEO-04, SEO-05, SEO-06, SEO-07, SEO-08

**Success Criteria** (what must be TRUE):
1. Google Rich Results Test validates RoofingContractor schema on all pages (not generic LocalBusiness)
2. Google Rich Results Test validates FAQPage schema on FAQ and service pages
3. Google Rich Results Test validates Article schema on all blog posts with proper author/date/image
4. Google Rich Results Test validates BreadcrumbList schema on all pages
5. Every page has unique meta title and description (verified: no duplicates in Google Search Console)
6. XML sitemap at /sitemap.xml with proper priority and changefreq values
7. Google Search Console shows all pages indexed with canonical URLs properly recognized

**Plans:** TBD

### Phase 4: Local SEO Hardening
**Goal:** NAP consistency, GBP alignment, and citation strategy ensure maximum visibility for "near me" and local searches.

**Depends on:** Phase 3 (technical SEO foundation must be solid before local signals)

**Requirements:** LOCAL-01, LOCAL-02, LOCAL-03, LOCAL-04

**Success Criteria** (what must be TRUE):
1. NAP (Name/Address/Phone) matches exactly across website, Google Business Profile, and all directory listings
2. LocalBusiness schema geo coordinates, hours, and areaServed match GBP exactly
3. Service page content includes local context references (Wilmington, coastal NC, hurricane zone)
4. Citation strategy document exists listing all directories with NAP verification checklist (Yelp, BBB, Angi, HomeAdvisor, Yellow Pages)

**Plans:** TBD

### Phase 5: Content Enrichment — Service Areas
**Goal:** Each area page has unique, locally-optimized content that ranks for "[service] in [city]" searches and avoids thin content penalties.

**Depends on:** Phase 4 (local SEO foundation must exist before area-specific content)

**Requirements:** AREA-01, AREA-02, AREA-03, AREA-04, AREA-05

**Success Criteria** (what must be TRUE):
1. Each of 8 area pages has 500+ words of unique content (verified: no duplicate paragraphs across pages)
2. User reading area page encounters local landmarks, neighborhoods, and building characteristics specific to that town
3. User sees area-specific testimonials and project photos where available (not generic content)
4. Area pages reference local building codes and coastal-specific concerns (hurricane prep, salt air, flood zones)
5. User can navigate from area pages to relevant service pages and blog posts via contextual internal links

### Phase 6: Content Enrichment — Storm/Insurance Blog
**Goal:** Blog content targets high-intent commercial keywords for storm damage, insurance claims, and Fortified certification to capture post-storm traffic.

**Depends on:** Phase 5 (area content establishes local authority before commercial blog content)

**Requirements:** BLOG-01, BLOG-02, BLOG-03, BLOG-04, BLOG-05, BLOG-06

**Success Criteria** (what must be TRUE):
1. User searching "hurricane roof damage insurance claim NC" finds Breeze Roofing blog post in top 10 results
2. Blog includes posts on storm damage, insurance claim process (NC-specific), Fortified certification benefits, and seasonal maintenance
3. User reading blog post sees contextual internal links to relevant service pages and estimate form (topic cluster model)
4. Each blog post has Article schema markup validated by Google Rich Results Test
5. User can discover related blog posts via recommendations section at bottom of each post
